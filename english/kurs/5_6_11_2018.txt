====================
CLI - текстовая связь междя компьютером и оператором.
техники использют при администрировании юникс-серверов с cli инструментами.
cli is more reliable than gui.

COMPRESSION - метод упаковки данных для экономии дискового пространства или время скачивания.
zip и mp3 это образцы двух наиболее общих алгоритма сжатия файлов.
every file can be compessed.


DEVICE DRIVER - ПО, которое преобразует данные, поступающие от компонента или периферийного устройства в данные, которые ОС может использовать.
ИТ-техники спрашивают конечного пользователя установил ли он недавние обновления и драйверы устройств.
usually drivers can be downloaded from manufacturers wibsite.


FILE PERMISSIONS - набор строгих правил для контроля чтения, записи и доступа на выполнение для файла или директории.
компания переключилась с фат32 на нтфс потому что первая не поддерживала права на файлы.
root user has all permissions.


FORMAT -приготовление устройства для хранения данных, стирая существующи данные.
необходимо отформатировать флэш-накопитель или жесткий диск, прежде чем он может использоваться для хранения данных.
in most cases formatting is quick procedure.


GUI - связь меджу компьютером и оператором на основе иконок.
большинство пользователей предпочитают гуи на основе иконок поверх интерфейса командной строки.
on server rarely uses a gui.


HIDDEN FILE - файл, который не отображается по умолчанию в списке директорий. обычно из соображений безопасности или из-за нежелания смущать пользователей. 
администратор юыл расстроен когда пользователь обнаружил способ отображить и удалить несколько срытых файлов.
hidden files is exist in all filesystems.


KERNEL - фундаментальная часть ОС, отвечающая за предоставление  доступа к железу машины.
системному администратору нужно обновлять ядро чтобы предоставить нативную поддержку последовательных ATA-дисководов.
without kernel OS can not work.


LINUX - опенсорсная версия юникса, разработанная командой добровольцев со всего мира.
линукс изначально был разработан линусом торвальдсом, который хотел юникс-подобную ОС, умеющую запускаяться на стандартном PC.
linus torvalds developed only kernel for linux, but linux itself richard stolman developed.


MULTITASKING - конкуретное выполнение двух или более задач процессором.
юникс это оригинальная мультизадачная  среда, спроектированная изначально для распределения ресурсов по сети.
true multitasking depends on the number of processors.


OS - гуи или кли ПО, связывающее компьютер и оператора. так же предоставляет фреймворк для повышения производительности, такой как офис, браузер или языки программирования.
программист тестирует свой ява-код во многих ОС , включая виндоуз, линукс, макос.
windows id the most popular OS at moment.


SIGN IN - ввод информации, связанной с именем учётной записи и паролем для получения доступа к ресурса компьютера. 
человек не может зарегистрироваться не его виндоуз-машине потому что он забыл своё имя пользователя и пароль. 
all users must sign in to access computer resources.


SIGN OUT - для завершения сессии с компьютером или ресурсами сети.
сервер был настроен на автоматическое разлогинивание  пользователей поле 10 минутной неактивности.
automatiс sign out helps to save resources.


VM - ПО, которое подражает производительности одного или более устройств для того чтобы запустить ПО независимо  от основного железа.
для того чтобы запустиь ява-программу в виндоуз необходимо скачать соответствующую вирт.машину.
there are many types virtual machines.


X - набор инструментов для юникс-систем, лежащий в основе многочисленных ГУИ-оконных менеджеров, включая КДЕ и ГНОМ.
человек был удивлён насколько схожа функциональность иксов с функциональностью микрософт виндоуз.
x-window used in unix-like OS.
